# خطة ميزات نظام Zman الجديدة

هذا الملف يمثل خطة العمل والمواصفات الفنية للميزات السبع الجديدة المطلوبة لنظام Zman.

## الميزات والتغييرات

---

### الميزة 1: إدارة أصناف المشتريات
- **الهدف:** استبدال حقل الإدخال الحر للأصناف بقائمة منتقاة مُدارة، مع السماح بالإدخال الحر.
- **التعديلات الفنية:**
  - إنشاء جدول `purchase_item_catalog` (uuid + name + timestamps + indexes + soft delete + triggers).
  - كتابة الـ CRUD كاملة (Server Actions + hooks).
  - تعديل `PurchaseForm.tsx` لعرض الأصناف المتاحة في قائمة منسدلة مع خيار "أخرى (إدخال يدوي)".
  - إضافة زر إدارة الأصناف بجانب شريط البحث في صفحة المشتريات.

---

### الميزة 2: إدارة فئات المصاريف
- **الهدف:** استبدال حقل الفئة في المصاريف بقائمة مُدارة.
- **التعديلات الفنية:**
  - إنشاء جدول `expense_category_catalog` (uuid + name + timestamps + indexes + soft delete + triggers).
  - كتابة الـ CRUD كاملة.
  - تعديل `ExpenseForm.tsx` لعرض الفئات المتاحة مع خيار الإدخال اليدوي.
  - إضافة زر إدارة الفئات بجانب شريط البحث في صفحة المصاريف.

---

### الميزة 3: تاريخ التسليم وتاريخ الاستلام في الطلبات
- **الهدف:** إضافة حقلين للتسجيل والفرز الزمني، وعرض تاريخ التسليم في التقويم.
- **التعديلات الفنية:**
  - إضافة `delivery_date` (nullable) و `received_date` (not null default now) لجدول `order`.
  - تحديث Zod schemas والـ queries والـ actions لتدعم هذين الحقلين.
  - تحديث `OrderCalendar.tsx` ومنافذ البحث لتعتمد على `delivery_date` بدلاً من تاريخ الإنشاء `created_at`.
  - تعديل `OrderForm` و`OrderDetail` لتعديل وعرض هذين الحقلين.

---

### الميزة 4: قالب رسالة تأكيد الزبون القابل للتعديل
- **الهدف:** تمكين تعديل قالب رسالة WhatsApp وتعبئة متغيرات العميل تلقائياً.
- **التعديلات الفنية:**
  - إنشاء جدول `message_template` (key primary key, template text, updated_at).
  - كتابة server action لجلب وتحديث القالب.
  - كتابة hook لاستدعاء وكاش القالب.
  - تعديل دالة `buildOrderWhatsAppLink` لتطبيق متغيرات القالب (`{customerName}`, `{productName}`, `{quantity}`, `{totalPrice}`, `{deliveryDate}`, `{notes}`).
  - إضافة زر محرر القالب بجانب البحث في صفحة الطلبات.

---

### الميزة 5: تفعيل زر حالة الطلب على بطاقة الطلب
- **الهدف:** تمكين تغيير حالة الطلب مباشرة من قائمة الطلبات.
- **التعديلات الفنية:**
  - استدعاء `useUpdateOrderStatus` داخل `OrderCard.tsx`.
  - استبدال ملصق الحالة الثابت بقائمة منسدلة (select dropdown) أنيقة.
  - تطبيق `e.stopPropagation()` لمنع انتقال الضغط لفتح تفاصيل الطلب.

---

### الميزة 6: رقم هاتف احتياطي للطلب
- **الهدف:** إضافة حقل هاتف بديل اختياري للعملاء.
- **التعديلات الفنية:**
  - إضافة `customer_phone_alt` لجدول `order`.
  - تحديث Zod schemas والـ form وتفاصيل العرض.

---

### الميزة 7: تحويل المبيعات وتأكيد الإيراد
- **الهدف:** توضيح تحويل المبيعات في الواجهة وحفظ السجل.
- **التعديلات الفنية:**
  - تغيير عنوان زر التحويل في `OrderDetail.tsx` إلى "تحويل إلى مبيعات (تسجيل إيراد)".
  - التأكد من تحديث كاش الأرقام والتقارير عند التحويل.
